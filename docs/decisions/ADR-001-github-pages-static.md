# ADR-001: GitHub Pages as Static Hosting

**Date:** 2026-03-15
**Status:** Accepted

---

## Decision

`aeothis-site` is hosted on GitHub Pages as a fully static site. There is no server-side rendering, no database, and no backend application. All dynamic behavior routes to external services (Formspree for chatbot form submission, the aeothis-tools webhook for pipeline processing).

---

## Context

The site serves two distinct purposes: public marketing and client delivery portal. The client delivery portal (proposal pages, guide pages, dashboards) is static HTML generated by tools in `aeothis-tools` and deployed by committing files to the repo.

Three hosting options were considered:

1. **GitHub Pages (static)** -- Free, reliable, no infrastructure to manage, instant deploys on push. All content is pre-generated by agent tools.
2. **VPS (dynamic server)** -- Requires maintenance, adds operational overhead, needed if server-side logic were required.
3. **Managed platform (Vercel, Netlify)** -- More capable but overkill for static output.

The proposal page encryption model is a key enabler of the static choice: content is AES-encrypted in the browser using a client-side key (the access code). No server-side session or auth layer is needed because the encryption itself is the access control. This means a static file host is sufficient for delivering confidential client content.

---

## Consequences

- All site changes are made by editing files locally and running `git push`. No server restarts, deploys, or build pipelines required.
- Client pages (proposal, guide, dashboard) are added by committing new HTML files under `clients/{slug}/` and pushing.
- The site does not support server-side logic. Features requiring dynamic computation must route through external services.
- CSS cache-busting is manual: update the `?v=` query param on `<link>` tags referencing `style.css` when the file changes.
- The repo is public. Client pages are accessible by URL but protected by client-side encryption (proposal pages) or obscurity (guide/dashboard pages linked only via email).
